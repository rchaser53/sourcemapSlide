sourcemapの出力結果

							
{
	"version" : 3,
	"file": "out.js",
	"sourceRoot": "",
	"sources": ["foo.js", "bar.js"],
	"sourcesContent": [null, null],
	"names": ["src", "maps", "are", "fun"],
	"mappings": "A,AAAB;;ABCDE;"
}

interfaceと概要

							
{
	version: 3;					// version 3なので3固定。
	file: string;					// ソースマップの対象ファイル
	sourceRoot?: string;			// 基準となるディレクトリ
	sources: string[]; 				// 対象ファイルの元となったファイル
	sourcesContent?: string[]; 		// 変換元となったソース
	names: string[];				// 対象ファイルに出力されている単語
	mappings: string; 				// 生成前のファイルとの位置関係
}

sourcesContent

変換前のソース。ts=>jsであればtsのファイルの内容が出力されている

sourcemapを使用している場合、ブラウザはfileから変換前のソースを参照しにいく

その際に対象のファイルが存在しないと、何も書かれていないファイルが表示される

sourcesContentに値が変換前のソースがあれば、優先して参照するので変換前のソースがなくても大丈夫
